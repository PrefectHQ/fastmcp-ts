title: "Tasks"
description: "Planned support for long-running tools with background execution and client polling"

---

FastMCP does not support tasks yet. This page describes planned work, not a current feature.

Tasks let a tool return a handle right away, run the handler in the background, and let the client poll for the result later. This helps with work that outlives a normal request, such as rendering a large report or training a model.

In MCP, tasks are an optional extension, not a core protocol feature. The extension is defined in a separate, experimental specification: [SEP-2663](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/2663) and the [`ext-tasks`](https://github.com/modelcontextprotocol/ext-tasks) reference repository. The MCP maintainers may still change this design, or drop it.

FastMCP will add task support once the extension stabilizes. Until then, use these tools for long-running work:

- **Multi round-trip requests.** A handler can ask the client for more input mid-call and resume once the client replies. See [Context](/servers/context).
- **Progress notifications.** A handler can send progress updates while it runs. See [Context](/servers/context).
- **Resource subscriptions.** A client can watch a resource and get notified when it changes. See [Resources](/servers/resources).

This page will describe the real API once tasks ship.
